### Live signal pipeline

1. `signal_log.generate_live_signals(cadence=...)` → `get_live_recommendation(include_wti=True)`
2. Live overlay: 15-min spread values from the recorder DB go in as `live_actuals`; live curve M1-M12 re-buckets the regime axis.
3. Per-spread `predict()` produces (point, p10/p50/p90, resid_std).
4. `model_health.check_cell` gates against extrapolating `fair_value` (`|point − rolling_mean| / rolling_std > 3`). When failed, `direction = NEUTRAL`, confidence = 0.
5. Sanity gate `_SANITY_Z_CAP = 8.0` drops any obviously misspec |z|>8 row (catches feature-stale extreme z's).
6. Session dedup: one OPEN row per `(instrument, direction)`; refires bump `bar_count`, direction flip closes with `flip`.
7. Perf updater every 15 min: marks at the **trigger level** on TP/SL (was previously at live `cur` — fixed); time-stop is **30 trading days** via `np.busday_count` (was calendar days — fixed); ambiguous bar resolves to `stop` (conservative); skips sweep entirely when feed is >90 min stale.

## 3. Known caveats (honest)

1. **Slow features (`inv_*`, `cot_*`, `realised_vol_20d`, `crack_*`, `real_rate`, `ovx_vix_ratio`) are 3+ weeks stale.** Daily file `/Data` last updated 2026-05-26. Live spread + live curve are intraday but the rest of the features carry yesterday's-daily values. Result: model fair_value can be high vs live spread today (z=-11 on brent_m1_m2 from the global model is a stale-feature artefact, NOT a model bug). The sanity gate at |z|>8 catches the worst.
2. **`wti_m1_m2` PF 0.76** — known synth-trained WTI offset. Fix is to retrain WTI on real daily settlements when an EOD file becomes available.
3. **`brent_m1_m2` row #6 + `brent_fly_123` row #5** in the live `signal_log` are CLOSED-out legacy from BEFORE the global model came online. They show `winner_model: ElasticNet/XGBoost` because they were opened by the old per-cell engine. New signals show `winner_model: LightGBM`.
4. **HF deployment** — code pushed; if HF doesn't auto-rebuild on a GitHub push, the user needs to click **Settings → Factory rebuild** in the HF UI (or `huggingface-cli upload <user>/pulse deploy/hf_space/Dockerfile Dockerfile --repo-type space` to trigger).
5. **sklearn version skew (1.7→1.9)** is moot for the global model — it uses LightGBM native API, not sklearn wrappers. The legacy pooled/composite pkls still trip the warning, but they're no longer the production path.
6. **Frontend hashed assets** updated each build — the `backend/static/assets/index-*.{js,css}` is regenerated by `npm run build`, so check `index.html` for the new asset references.
